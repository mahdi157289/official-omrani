enum UserRole {
  CUSTOMER  // Regular customers
  EDITOR    // Store employees - can manage products/content
  ADMIN     // Store owner - full system access
}

enum UserStatus {
  UNCLAIMED  // Account created via guest checkout, not yet activated
  ACTIVE     // Active account
  INACTIVE   // Deactivated account
  SUSPENDED  // Temporarily suspended
}

enum OrderStatus {
  PENDING           // Order placed, awaiting confirmation
  CONFIRMED         // Order confirmed by store
  PREPARING         // Being prepared in kitchen
  READY_FOR_PICKUP  // Ready for customer pickup
  OUT_FOR_DELIVERY  // On delivery route
  DELIVERED         // Successfully delivered
  CANCELLED         // Order cancelled
  REFUNDED          // Order refunded
}

enum DeliveryType {
  PICKUP_KAIROUAN  // Pickup from Kairouan store
  PICKUP_TUNIS     // Pickup from Tunis store
  DELIVERY         // Home/office delivery
}

enum PaymentMethod {
  CASH_ON_DELIVERY  // Cash payment upon delivery
  BANK_TRANSFER     // Bank transfer (future)
  CARD              // Credit/debit card (future)
}

enum PaymentStatus {
  PENDING     // Awaiting payment
  PROCESSING  // Payment being processed
  COMPLETED   // Payment successful
  FAILED      // Payment failed
  REFUNDED    // Payment refunded
}

enum ProductStatus {
  DRAFT       // Not visible to customers
  ACTIVE      // Available for purchase
  OUT_OF_STOCK // Temporarily unavailable
  ARCHIVED    // No longer sold
}

enum MediaType {
  IMAGE
  VIDEO
  DOCUMENT
}

# Database Schema - Summary & Usage Guide

## 🗃️ Models Overview

| Model | Purpose | Key Features |
|-------|---------|--------------|
| **User** | Customer & admin accounts | Multi-role system, guest checkout support, multilingual preferences |
| **Address** | Delivery addresses | Default address flag, geolocation support, multi-address per user |
| **Product** | Product catalog | Full multilingual support, inventory tracking, SEO fields |
| **ProductVariant** | Product variations (weights/sizes) | Weight-based pricing, separate inventory tracking |
| **Category** | Product categorization | Hierarchical categories, featured categories |
| **Media** | Images & files | Cloudinary integration, polymorphic relations |
| **Order** | Customer orders | Comprehensive status tracking, delivery scheduling |
| **OrderItem** | Items within orders | Price snapshot, variant linking |
| **Event** | Promotions & events | Date-based, product associations |
| **Review** | Product reviews | Rating system, moderation flags |
| **WishlistItem** | Customer wishlists | User-product unique constraint |
| **SiteConfig** | Dynamic configuration | JSON-based flexible values, multilingual labels |
| **DeliveryZone** | Delivery areas | Zone-based pricing, scheduling rules |
| **PageView** | Analytics tracking | Visitor behavior, device detection |
| **ActivityLog** | Admin activity audit | Change tracking, user action logging |

## 🔗 Key Relationships

### 1-to-Many Relationships:
- **User → Addresses** (One user, multiple addresses)
- **User → Orders** (One user, multiple orders)
- **Category → Products** (One category, multiple products)
- **Product → Variants** (One product, multiple weight variants)
- **Order → OrderItems** (One order, multiple items)

### Polymorphic Media:
Media files can be attached to:
- Products (product images)
- Categories (category images)
- Events (event banners)

### Many-to-Many:
- **Events ↔ Products** (Events can feature multiple products)

## 📊 Analytics & Reporting

The schema supports comprehensive analytics:

1. **Sales Analytics**: Order status tracking, revenue calculation
2. **User Analytics**: Page views, session tracking
3. **Inventory Analytics**: Stock levels, best sellers
4. **Activity Auditing**: Admin change logging
